# テストシナリオ: sound-converter 全機能

> 作成日: 2026-04-04
> 対象機能: ファイルアップロード・無音トリミング・音量正規化/リミット・再生速度調整・波形プレビュー・ダウンロード
> 対象ページ: `/`（シングルページアプリケーション）
> 前提条件: 認証不要。ブラウザは SharedArrayBuffer 対応（Chrome 92+ / Firefox 79+ / Edge 92+）。開発サーバーは `npm run dev` で起動済み（`http://localhost:5173`）。テスト用音声ファイル（WAV / MP3 / OGG、各種無音パターン）を手元に用意しておくこと。

## 1. ユニットテスト: detectSilence

> 対象ファイル: `src/utils/audio/detectSilence.ts`
> テストファイル: `tests/utils/detectSilence.test.ts`
> モック: `AudioContext` は `vi.mock` で代替。合成 ArrayBuffer は `tests/fixtures/` から読み込む。

| # | テストケース名 | 入力条件 | 期待される戻り値 |
|---|--------------|---------|----------------|
| U-01 | 先頭のみ無音 | 先頭 500ms が -60dBFS（閾値 -50dBFS 以下）、残りは -10dBFS の合成 PCM | `startTrimSec` ≈ 0.5、`endTrimSec` ≈ 0.0（誤差 ±10ms 以内） |
| U-02 | 末尾のみ無音 | 末尾 500ms が -60dBFS、先頭・中間は -10dBFS の合成 PCM | `startTrimSec` ≈ 0.0、`endTrimSec` ≈ 0.5（誤差 ±10ms 以内） |
| U-03 | 先頭・末尾両方に無音 | 先頭 300ms + 末尾 400ms が -60dBFS の合成 PCM | `startTrimSec` ≈ 0.3、`endTrimSec` ≈ 0.4（誤差 ±10ms 以内） |
| U-04 | 無音なし（トリミング不要） | 全区間が -10dBFS の合成 PCM | `startTrimSec` = 0.0、`endTrimSec` = 0.0 |
| U-05 | 全体が無音 | 全区間が -80dBFS の合成 PCM | `startTrimSec` = ファイル総長（秒）、`endTrimSec` = 0.0、または `{ startTrimSec: 0, endTrimSec: 0 }` のいずれか実装仕様に従う（全無音として処理を中断し、エラーを投げる実装の場合はそのエラーを確認） |
| U-06 | MIN_SILENCE_DURATION 未満の無音は無視 | 先頭に 50ms（< 100ms のデフォルト閾値）の無音、残りは -10dBFS | `startTrimSec` = 0.0（短すぎる無音は無視される） |
| U-07 | カスタム閾値: silenceThreshold を -30dBFS に設定 | 先頭 500ms が -40dBFS（-30dBFS より小さいが -50dBFS より大きい）の合成 PCM | `startTrimSec` ≈ 0.5（-40dBFS は -30dBFS 以下なので無音と判定） |
| U-08 | カスタム閾値: silenceThreshold を -70dBFS に設定 | 先頭 500ms が -60dBFS の合成 PCM | `startTrimSec` = 0.0（-60dBFS は -70dBFS より大きいので無音と判定されない） |
| U-09 | ステレオ音声（2ch）の先頭無音検出 | 2チャンネル PCM で先頭 200ms が無音 | `startTrimSec` ≈ 0.2（チャンネル平均 RMS で判定されること） |
| U-10 | minSilenceDuration をカスタム設定（200ms）で短い無音を検出 | 先頭 150ms が無音（デフォルト 100ms より長く、200ms より短い） | minSilenceDuration=100ms なら `startTrimSec` ≈ 0.15、minSilenceDuration=200ms なら `startTrimSec` = 0.0 |
